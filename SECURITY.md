# Security Policy

SliceOps is an open framework maintained by Andrés Ramírez Sierra. This policy
covers the SliceOps GitHub organization and all its repositories (`spec`,
`toolkit`, `sliceops-website`) unless a repository ships its own `SECURITY.md`.

## Reporting a vulnerability

Please do **not** open a public issue for security problems.

- **Preferred:** GitHub **private vulnerability reporting** — the *"Report a
  vulnerability"* button under each repository's **Security** tab.
- **Or email:** **privacy@sliceops.org** with a description, the affected
  repository/version, and reproduction steps.

We aim to acknowledge within **5 business days** and to agree a disclosure
timeline with you. Please allow reasonable time to remediate before any public
disclosure (coordinated disclosure).

## Scope

**In scope:** the framework specification, the toolkit (validators / CI
templates), and the website (sliceops.org) source.
**Out of scope:** third-party runtimes that implement SliceOps, and unsolicited
messages to the public support chat (inherent to any public chat widget).

## Supported versions

The latest published release of each repository is supported — see each repo's
**Releases** page.

*This is a project security policy, not legal advice.*
