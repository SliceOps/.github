# SliceOps™

**The open framework and audit plane for AI-first software engineering.** Multi-agent teams ship auditable software — not vibe code.

AI can write the code. SliceOps governs the part that matters: **the decisions**. Every change is an atomic *slice*, every decision is a recorded, human-authored artifact, and the result is auditable *by construction* — across **12 canonical principles** (Why → How → What).

## What makes it different

- **Decision integrity by construction** — when agents do the building, the decisions are what you actually ship; SliceOps makes them atomic, auditable, and human-authored.
- **An audit plane** — a governed home for every decision, so compliance evidence already exists.
- **Context Discipline (P12)** — agents are memoryless; the corpus is their shared, single source of truth.
- **Platform-agnostic** — runs on any text-based AI agent + git. No runtime lock-in.
- **Dogfooded** — these repos run the framework's own gates (validators, tests, security) in CI on protected branches.
